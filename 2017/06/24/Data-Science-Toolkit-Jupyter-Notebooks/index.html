Data Science Toolkit: Jupyter Notebooks

Jun 24, 2017

The Ability to rapidly prototype and test out new ideas often determines whether or not any project will eventually be successful. This is even more true in Data Centric Software Development. Therefore, as an engineer, it’s important to have tools that encourage and enable your ability to quickly move through the prototype - test - re-prototype feedback loop. Jupyter Notebooks is one such tool.

What

Jupyter Notebooks is a browser based development environment that grew out of the IPython project. It was developed with numerical programming in mind; It mainly supports Julia, Python and R. In this blogpost, I walk through how to locally install and customize it.

Installation.

The basic assumption for this section is that you are working on a *nix machine with Python (and Pip, the package manager) installed.

First, run $ pip install jupyter. This should fetch the latest version of Jupyter available in PyPi. If you are using Python 3.x +, you’ll need to use this command instead: $ pip3 install jupyter.
That’s it. Jupyter should now be globally available on your system. If all is well, typing

should open a new tab in your web browser pointing to a running local instance of Jupyter. If your system complains that the jupyter command was not found, then the installation wasn’t successful.

customization

Vanilla jupyter is great. However, we can make it even prettier. My customization tool of choice is jthemes. The link points to the project’s github repository which has comprehensive documentaton and guidelines on how to install and use jthemes. If you have jupyter, I encourage you to check it out, I’m positive that you will appreciate it.

Best practices.

This last section is somewhat underdeveloped. In fact, it is a work in progress. I intend to flesh this out into an independent blogpost about best practices when using Jupyter. What follows are rudiments of the said blogpost. Feel free to add some of your best practices in the comment section.

  1. Isolate Memory or network intensive code into an independent cell. Code that feteches some resources over the network or reads into memory some large (data) file should be decoupled from code that uses the fetched resource. This ensures that such resources, which are often static, are only read into memory once. Therefore, you can tinker around with the logic taht relies on the resource without having to wait around for the resource to be fetched again.
